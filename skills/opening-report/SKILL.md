# 开题报告撰写（Opening Report）

你现在要帮用户撰写学位论文开题报告。开题报告是学位论文工作的起点，必须结构完整、论述严谨、格式规范。

## 适用场景

- 硕士研究生开题报告
- 博士研究生开题报告
- 本科毕业设计开题报告

## 开始前必须确认

使用 `ask_user` 工具确认以下信息：

1. **学位层次**：硕士 / 博士 / 本科
2. **学校/学院**：不同学校有不同模板要求
3. **研究方向**：用户的研究领域和选题
4. **输出格式**：Markdown / LaTeX / Word
5. **是否有学校模板**：如果用户有学校提供的 LaTeX 或 Word 模板，优先适配

## 标准开题报告结构

### 硕士开题报告（推荐结构）

```
1. 选题背景与意义
   1.1 研究背景
   1.2 研究意义（理论意义 + 应用价值）

2. 国内外研究现状
   2.1 [主题一]研究现状
   2.2 [主题二]研究现状
   2.3 现有研究的局限性（文献 gap）

3. 研究目标与内容
   3.1 研究目标
   3.2 研究内容（分条列出，通常 3-4 个子问题）

4. 研究方案与技术路线
   4.1 整体研究框架（建议画架构图）
   4.2 关键技术/方法
   4.3 实验设计与验证方案

5. 预期创新点（2-3 个）

6. 研究进度安排（表格形式）

7. 参考文献（不少于 30 篇，近 5 年文献占比 > 50%）
```

### 博士开题报告（推荐结构）

```
1. 选题背景与意义
   1.1 研究背景
   1.2 理论意义
   1.3 应用价值

2. 国内外研究现状（文献综述）
   2.1 [方向一]研究进展
   2.2 [方向二]研究进展
   2.3 [方向三]研究进展
   2.4 研究现状评述与不足

3. 研究目标与科学问题
   3.1 总体研究目标
   3.2 拟解决的关键科学问题（2-3 个）

4. 研究内容与方案
   4.1 研究内容一：[标题]
       - 研究思路
       - 技术方案
       - 预期成果
   4.2 研究内容二：[标题]
       ...
   4.3 研究内容三：[标题]
       ...
   4.4 技术路线图

5. 预期创新点与贡献（3-4 个）

6. 研究基础与条件
   6.1 前期工作基础
   6.2 实验条件与数据来源

7. 研究进度安排

8. 参考文献（不少于 50 篇，近 5 年文献占比 > 60%）
```

## 各章节写作指南

### 1. 选题背景与意义

**写法**：从大到小，从宏观到具体。

- **研究背景**：介绍领域大背景 → 聚焦到具体问题 → 指出现有不足
- **研究意义**：
  - 理论意义：本研究对XX理论/方法的推进
  - 应用价值：解决什么实际问题，有什么落地场景

**常见问题**：
- 背景写得太泛，像百科全书 → 应聚焦到研究问题
- 意义写得太空，全是大话 → 应具体说明"做了这个研究之后能怎样"

### 2. 国内外研究现状

**写法**：不是文献堆砌，是有逻辑的综述。

- 按主题/方向分组，不要按时间顺序流水账
- 每个方向：代表性工作 → 方法特点 → 优缺点
- 最后必须有"综合评述"段：总结现有工作的共同不足，引出你的研究动机

**引用要求**：
- 每段观点都要有引用支撑
- 优先引用近 5 年的高质量期刊/会议论文
- 中英文文献都要覆盖（国内外研究现状）
- 格式统一：文中 `[1]` 或 `(Author, Year)`，文末参考文献列表

**常见问题**：
- 只罗列不评价 → 每篇文献要说"它做了什么、贡献是什么、局限在哪"
- 没有综合评述 → 缺少从现有文献到你的研究的逻辑桥梁

### 3. 研究目标与内容

**写法**：

- **研究目标**：一句话总目标 + 2-3 个子目标
- **研究内容**：对应子目标，每条清晰描述要"做什么"和"怎么做"
- 研究内容之间要有逻辑递进关系，不是互不相关的几件事

**常见问题**：
- 目标和内容混淆 → 目标是"要达到什么效果"，内容是"具体做什么事"
- 研究内容太多太散 → 硕士 3-4 个子内容为宜

### 4. 研究方案与技术路线

**写法**：

- **整体框架图**：用文字描述架构，说明各模块关系（建议用 ASCII 或 Mermaid 图）
- **关键技术**：每个技术点说清楚"为什么用这个方法"和"具体怎么做"
- **实验设计**：数据集、评估指标、对比方法、消融实验

**常见问题**：
- 方案太笼统 → 应该具体到"用什么算法/模型/数据"
- 没有验证方案 → 必须说明怎么证明你的方法有效

### 5. 预期创新点

**写法**：

- 每个创新点一句话概括 + 1-2 句解释
- 创新点要和研究内容对应
- 区分"方法创新"和"应用创新"

**常见问题**：
- 写成"首次提出XXX" → 除非你确定没人做过，否则用"提出一种新的XXX方法/视角"
- 创新点和研究内容对不上 → 每个创新点应该能在研究内容中找到对应

### 6. 研究进度安排

**格式**：表格

```markdown
| 阶段 | 时间 | 主要工作 |
|------|------|----------|
| 文献调研与开题 | 第1-2月 | 完成文献调研，撰写开题报告 |
| 系统/方法设计 | 第3-4月 | 完成方案设计 |
| 核心实现 | 第5-8月 | 实现核心方法/系统 |
| 实验验证 | 第9-12月 | 设计实验，完成验证 |
| 论文撰写 | 第13-16月 | 撰写论文初稿 |
| 修改与答辩 | 第17-18月 | 论文修改，准备答辩 |
```

- 硕士通常 18-24 个月，博士 36-48 个月
- 阶段之间可以有重叠
- 要留出论文修改和答辩准备的时间

### 7. 参考文献

**格式**：优先使用 GB/T 7714 格式（中国标准），除非用户指定其他格式。

```
[1] 作者. 题名[J]. 刊名, 年, 卷(期): 起止页码.
[2] Author A, Author B. Title[J]. Journal, Year, Vol(No): Pages.
[3] 作者. 书名[M]. 出版地: 出版社, 年.
[4] Author. Title[C]//Conference Name. Year: Pages.
```

**数量要求**：
- 本科：不少于 15 篇
- 硕士：不少于 30 篇
- 博士：不少于 50 篇
- 近 5 年文献占比应超过 50%

## 工作流程

### Step 1: 信息收集

用 `ask_user` 确认：
- 学位层次（硕士/博士/本科）
- 学校（影响模板格式）
- 研究方向和选题
- 输出格式（Markdown/LaTeX/Word）
- 是否有学校模板

### Step 2: 建立 TASK.md

在 `output/TASK.md` 创建详细 checklist，示例：

```markdown
# 当前任务：撰写硕士开题报告

## 目标
产出一份完整的硕士开题报告（Markdown 格式），包含 7 个标准章节

## 步骤
- [ ] 确认用户信息和格式要求
- [ ] 撰写第1章：选题背景与意义
- [ ] 撰写第2章：国内外研究现状
- [ ] 撰写第3章：研究目标与内容
- [ ] 撰写第4章：研究方案与技术路线
- [ ] 撰写第5章：预期创新点
- [ ] 撰写第6章：研究进度安排
- [ ] 整理参考文献
- [ ] 全文检查与格式调整
```

### Step 3: 逐章撰写

- 按 TASK.md 的步骤逐章写
- 每章写完更新 TASK.md
- 所有内容写到 `output/` 目录的文件中

### Step 4: 全文检查

- 各章节逻辑衔接
- 术语和表述一致性
- 参考文献完整性和格式
- 字数是否达标

## 产出物

保存到 `output/` 目录：
- `output/OPENING_REPORT.md` — Markdown 版本
- `output/OPENING_REPORT.tex` — LaTeX 版本（如果用户需要）
- `output/OPENING_REPORT.pdf` — 编译后的 PDF（如果有 LaTeX 环境）

## 质量标准

- 每章都有实质内容，不是空架子
- 文献引用真实、可查证
- 研究内容之间有逻辑关系，不是独立的几件事拼凑
- 技术方案要具体到"用什么方法解决什么问题"
- 创新点要和研究内容对应
- 全文字数：硕士 8000-15000 字，博士 15000-30000 字
